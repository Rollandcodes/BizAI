# BizAI Development Instructions

BizAI is an AI-powered customer service assistant SaaS for local businesses. This file documents the project structure, conventions, and key implementation guidelines.

## Project Overview

**Purpose:** Provide local businesses with an AI-powered chat widget that handles customer inquiries automatically, integrated with PayPal for subscription management and Supabase for data persistence.

**Tech Stack:**
- **Frontend:** Next.js 14 with App Router, TypeScript, Tailwind CSS
- **Backend:** Next.js API Routes
- **Database:** Supabase (PostgreSQL)
- **AI:** OpenAI GPT-4
- **Payments:** PayPal
- **Authentication:** Supabase Auth (to be implemented)

## Key Conventions

### 1. Component Structure
- All components that use React hooks are marked with `'use client'` directive
- Components in `components/` are reusable across pages
- Use TypeScript interfaces for component props and data structures
- Style components with Tailwind CSS utility classes

### 2. API Routes
- Located in `app/api/` following Next.js App Router conventions
- Return `NextResponse` with appropriate status codes
- Include error handling with try-catch blocks
- Document TODOs for incomplete implementations

### 3. Type Definitions
- Define types in the same file where they're used (`supabase.ts`, `openai.ts`)
- Export types for reuse across the app
- Use TypeScript interfaces over types for object shapes

### 4. Environment Variables
- All sensitive keys stored in `.env.local`
- Public keys prefixed with `NEXT_PUBLIC_` (accessible in browser)
- Private keys used only in API routes and server utilities

### 5. Business Niches
- Supported niches defined in `lib/niches.ts`
- Each niche has a specific prompt template to guide AI responses
- Niches determine the context for AI customer service assistance

## Development Workflow

1. **Environment Setup**
   ```bash
   npm install
   # Copy .env.local template and fill in API keys
   ```

2. **Database Setup (Supabase)**
   - Create tables: `businesses`, `conversations`, `messages`
   - Set up Row Level Security (RLS) policies
   - Run migrations

3. **API Implementation**
   - Complete POST/GET endpoints in `app/api/`
   - Connect to Supabase for persistence
   - Integrate OpenAI for AI responses
   - Add PayPal payment handling

4. **Frontend Development**
   - Build and test components in isolation
   - Use Tailwind CSS for consistent styling
   - Test API integration with real endpoints

## Key Integration Points

### OpenAI Chat Endpoint (`/api/chat`)
- Receives user messages with business context
- Uses `lib/openai.ts` helper functions
- Returns AI-generated responses
- Should maintain conversation history for better context

### PayPal Integration (`/api/paypal/*`)
- Create order endpoint for plan selection
- Capture order endpoint after approval
- Store transaction data in Supabase
- Map plans to feature sets

### Business Settings (`/api/business`)
- POST: Save business details (name, niche, description)
- GET: Retrieve business configuration
- All data persisted in Supabase

### Embeddable Widget (`public/widget.js`)
- Injected into customer websites
- Loads chat interface in iframe
- Communicates with `/api/chat` for messages
- Customizable position and theme
